👤

Huésped o interesado

🤖

IA / Formulario (futuro)

Interpreta la consulta y la traduce a llamadas estructuradas. No toma decisiones críticas.

Verifica disponibilidad, crea consultas, registra pre-reservas, valida pagos y confirma reservas con reglas fijas.

Google Sheets — base de datos operativa

RESERVAS, DISPONIBILIDAD_CACHE, TARIFAS, CONSULTAS. Fuente de verdad única y auditable por cualquier persona del equipo.

👥

Franco, Rodrigo y Vicky ven el estado real en tiempo real. Jennifer recibe el calendario de limpieza.

db_recalcular_disponibilidad · v8
Motor de Disponibilidad

Regenera completa la tabla DISPONIBILIDAD_CACHE para todas las cabañas y fechas. Se ejecuta automáticamente después de cada operación que afecte disponibilidad. Garantiza que nunca haya datos desincronizados.

Recalcula todo Sin parámetros extra Fuente única
Validado ✓
db_crear_consulta · v3
Registro de Consulta

Cuando alguien pregunta por disponibilidad, este workflow registra la consulta o recupera una consulta activa existente. Evita duplicados y deja trazabilidad de todos los interesados, incluso los que no reservan.

Anti-duplicados Historial Trazabilidad
Validado ✓
db_crear_prereserva · v2
Pre-reserva Temporal

Crea una pre-reserva con fecha de vencimiento, verificando disponibilidad en dos capas antes de bloquear las fechas. Si hay conflicto, rechaza la operación. Llama automáticamente al motor de disponibilidad al finalizar.

El proceso completo

Flujo de una reserva

💬

Consulta

El huésped pregunta por fechas y cabaña por WhatsApp, web o bot.

📋

Registro

El sistema registra la consulta y verifica disponibilidad real.

🔒

Pre-reserva

Se bloquean las fechas temporalmente. El huésped tiene tiempo para pagar.

💳

Pago

(Próximamente) El huésped reporta el pago. El sistema re-valida disponibilidad.

✅

Confirmación

(Próximamente) La reserva se confirma. Calendario y equipo notificados automáticamente.
